Model Details

Each stream is independently trained with identity-aware validation and contributes orthogonal signal to the ensemble.

01

rPPG Physiological

Biological blood-flow signals extracted from 9 facial ROIs — deepfakes lack coherent rPPG patterns.

Face DetectionMediaPipe FaceMesh 468 lm
Signal MethodCHROM rPPG
Bandpass0.7–4.0 Hz (Butterworth 3rd)
Feature Dim117 (spectral + HRV + geometry)
ClassifiersXGBoost · LightGBM · HistGB
Meta-LearnerLogisticRegression (5-fold OOF)
CV StrategyGroupShuffleSplit 80/20
Output:rppg_predictions.csv → P_rPPG
02

EfficientNet-B4

ImageNet-pretrained spatial encoder with BiLSTM temporal modelling and multi-head self-attention.

BackboneEfficientNet-B4 (1792-dim)
Input16 × 224² · 5-fold StratGroupKFold
TemporalBiLSTM 2L×256H + 4-head MHA
LossFocal α=0.6 γ=2.0 · Label smooth 0.1
LR5×10⁻⁵ · Warmup 10% · CosineAnneal
SWAEpoch 30 · SWA-LR 5×10⁻⁵
TTA5 passes (flip · brightness · blur)
Output:cnn_predictions.csv → P_CNN
03

Xception + Frequency

Dual-branch: Xception spatial stream plus parallel DCT/FFT frequency branch capturing compression artifacts.

BackboneXception (2048-dim) + ECA Attn
Freq BranchLinear 2048→256-dim parallel
Fused Dim768-dim (temporal + freq)
CurriculumMixUp β=0.2 + CutMix α=1.0 (50/50)
Hard MiningWeightedRandomSampler · refresh/5ep
SWAEpoch 15 · manual BN update loop
TTA6 passes incl. center-crop 93%
Output:cnn_predictions.csv → P_CNN
04

Swin Transformer + DCT

Hierarchical shifted-window attention with on-the-fly DCT frequency branch and full 5-fold OOF calibration.

BackboneSwin-Tiny patch4_window7 (768-dim)
DCT Branch8×8 blocks → 128-dim · log|DCT|
Fused Dim704-dim (512 temporal + 192 freq)
CurriculumProgressive 5→10→16 frames/video
LSTMpack_padded_sequence · Xavier + Ortho init
SWAEpoch 15 · cosine anneal 5ep/group
CV5-fold OOF (all folds, one session)
Output:cnn_predictions_swin_oof_MASTER.csv
🔒
Identity-Aware Anti-Leakage Design

Every model uses StratifiedGroupKFold where groups are person identities extracted from filenames. Zero identity overlap between train and validation is verified by assertion before every training run — the primary source of inflated metrics in deepfake detection literature.
